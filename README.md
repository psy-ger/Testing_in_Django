# Testing_in_Django

Опис
---
Цей репозиторій містить простий Django-проєкт для демонстрації тестування форм і сериалізаторів за допомогою `pytest` та `pytest-django`.

У проєкті реалізовано додаток `tasks` з моделлю `Task`, формою для створення задач та сериалізаторами для REST-представлення.

Вимоги
---
- Python 3.8+
- virtualenv (рекомендовано)
- залежності в `requirements.txt` (`Django`, `djangorestframework`, `pytest`, `pytest-django`)

Швидкий старт (zsh)
---
1. Створіть та активуйте віртуальне середовище:

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Встановіть залежності:

```bash
pip install -r requirements.txt
```

3. Виконайте міграції:

```bash
python manage.py migrate
```

4. (Опціонально) Створіть суперкористувача для доступу в адмінку:

```bash
python manage.py createsuperuser
```

Запуск тестів
---
Для запуску тестів використовуйте `pytest`:

```bash
pytest -q
```

Файли з тестами знаходяться в `tasks/tests/` і включають тести для:
- форми `TaskForm` (перевірка валідності, порожні обов'язкові поля, перевірка `due_date`),
- сериалізаторів `TaskSerializer` і `TaskWithUserSerializer` (включно з кастомною валідацією `due_date` та вкладеним сериалізатором користувача).

Структура проекту (коротко)
---
- `manage.py` — CLI для Django
- `testing_in_django/` — налаштування проєкту
- `tasks/` — додаток з моделями, формами, сериалізаторами і тестами

Поради
---
- Якщо ви отримуєте проблеми з залежностями або середовищем, переконайтесь, що активували віртуальне середовище і інсталювали пакети з `requirements.txt`.
- Для локальної розробки можна запускати сервер:

```bash
python manage.py runserver
```
